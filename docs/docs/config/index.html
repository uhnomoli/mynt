The format of posts.url.

Variable Description
<day> Day of the post as a decimal.

Frontmatter attributes containing string values can be used in URL formats. For example, if all of your posts have an author attribute you could use it in your URL format like so: /<author>/<slug>/.

pygmentize: True

Whether or not code highlighting is enabled.

You can highlight code in Markdown by using fenced code blocks.

```html
<p>highlight me</p>
```

Or you can highlight code in HTML by simply setting the data-lang attribute of the <code> tag.

<pre><code data-lang="html">highlight me</code></pre>

You'll probably want a stylesheet for your highlighted code. You can read more about styles in Pygments here if you want to make your own, or you can grab one of the stylesheets included with Pygments from here.

filetypes: {}

A mapping of parsers to extensions. If more than one available parser supports an extension, you can explicitly define which is used here. The key is the name of the parser and the value is a list of extensions.

For example, to use the Python-Markdown parser over the default (Hoep):

filetypes:
  markdown:
    - md
    - markdown
tag_layout: None

The template used to generate tag pages.

Must be set to enable automatic tag page generation.

tags_url: /
The base URL for tag pages.

Parser-specific Options

Most parsers support optional features. To configure these, add a section with the parser's name, followed by a dictionary of options.

For example, to disable footnotes, and enable the underline extension for the default parser:

hoep:
  extensions:
    footnotes: False
    underline: True

To tell the Python-Markdown parser to output HTML5, and disable line numbers in code blocks:

markdown:
  output_format: html5
  extensions:
    - extra
    - codehilite
  extension_configs:
    codehilite:
      linenums: False

See the documentation for your chosen parser for available options.
